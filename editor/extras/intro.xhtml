AXEL Template-driven Document Editor

This application can be run either from a web server or directly from your local file system (URL starting with file:///). It is targeted first at XTiger XML document template creators so that they can test their templates.

If you open the editor from the local file system you need to authorize access to local URL from the XMLHTTPRequest object. For Firefox type about:config in the URL bar then click on security.fileuri.strict_origin_policy to set it to false (source: mozillazine.org). We didn't find how to do it with other browsers, so if you know let us know.

If you open the editor from a web server (URL starting with http://), including a WebDav server, it should work on most browsers.

  • Template file : enter a file path relative to the editor's location, or an absolute path to select a template file
  • Preselection : select directly a predefined template file
  • [Transform] : generate the document editor from the selected template file
  • [Source] : display the template source code inside a new window (Firefox only)
  • [Schema] : extracts a schematic representation of the template target data model (to be used on an empty document)
  • [Preview] : hide / show user interface controls to edit document structure
  • [Dump] : dump the XML content of the current document inside a popup window
  • [Download] : download the document data to your download folder (HTML5 feature not on all browsers)
  • [Input] : enter XML data from a popup window to the current document
  • [Load] : prompt a dialog to select a local XML file name and load data from it (HTML5 feature not on all browsers)

The Preferences pannel adds more commands :

  • Template folder [Set] : enter the path of a folder to include its template files in the preselection popup as #yours#
  • Output file [Write] [Read]: saves/load document data using Ajax, it may work or not depending on the browser / version and access mode (file system or web server), for test purpose you can open the application from the test server in scripts/server.

The preselection templates are defined inside the templates/shared/templates.js file of the distribution.

NOTE: the template file URL must have the same origin as the editor itself because of cross-domain access restrictions in web browsers. The reason is that the AXEL library inside the editor must be granted access to the iframe that contains the template file.
